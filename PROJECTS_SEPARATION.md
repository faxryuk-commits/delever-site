# Разделение проектов Delever

Проекты полностью разделены на два независимых репозитория.

## Структура проектов

### 1. Основной сайт (delever-site)
**Путь:** `/Users/faxriddin/delever site`  
**Репозиторий:** `https://github.com/faxryuk-commits/delever-site.git`  
**Домен:** `delever.io` (или `www.delever.io`)

**Содержит:**
- Публичные страницы сайта
- Компоненты интерфейса
- Страницы продуктов, интеграций, партнеров
- Футер и хедер

**Не содержит:**
- Админ-панель
- Аутентификацию
- Управление контентом

### 2. Админ-панель (delever-admin)
**Путь:** `/Users/faxriddin/delever-admin`  
**Репозиторий:** `https://github.com/faxryuk-commits/siteadmin.git`  
**Домен:** `admin.delever.io` (или отдельный поддомен)

**Содержит:**
- Визуальный редактор контента
- Управление страницами
- Аутентификацию
- Dashboard

**Не содержит:**
- Публичные страницы
- Компоненты основного сайта

## Деплой

### Основной сайт
- **Vercel проект:** `delever-site`
- **Build Command:** `npm run build`
- **Output Directory:** `dist`
- **Framework:** Vite

### Админ-панель
- **Vercel проект:** `siteadmin`
- **Build Command:** `npm run build`
- **Output Directory:** `dist`
- **Framework:** Vite

## Преимущества разделения

✅ **Независимость**
- Разные команды могут работать параллельно
- Независимые деплои
- Разные версии зависимостей

✅ **Безопасность**
- Админка не доступна на основном домене
- Разные методы аутентификации
- Легче ограничить доступ

✅ **Производительность**
- Меньший размер бандла для публичного сайта
- Разные настройки кеширования
- Оптимизация под конкретные задачи

✅ **Масштабируемость**
- Можно использовать разные технологии
- Независимое версионирование
- Легче добавлять новые функции

## Связь между проектами

Админка и сайт обмениваются данными через:
- **API** (REST или GraphQL)
- **Shared database**
- **Headless CMS**
