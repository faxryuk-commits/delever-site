# Решение проблемы 401 для статических ресурсов

## Проблема

Статические ресурсы (JS, CSS файлы) загружаются без параметров `admin_token` и `x-vercel-protection-bypass`, поэтому Vercel Authentication блокирует их с ошибкой 401.

## Решение

### Вариант 1: Использовать Protection Bypass через заголовок (рекомендуется)

Protection Bypass можно использовать не только через query параметр, но и через HTTP заголовок. Но для iframe это сложно.

### Вариант 2: Настроить исключения для статических файлов

В Vercel можно настроить исключения для определенных путей. Но это требует Pro план.

### Вариант 3: Отключить Vercel Authentication для статических файлов

Если Protection Bypass не работает для всех запросов, нужно либо:
1. Отключить Vercel Authentication полностью
2. Или использовать другой подход

### Вариант 4: Использовать Service Worker для добавления параметров

Можно использовать Service Worker в iframe для перехвата запросов и добавления параметров. Но это сложно.

## Рекомендация

**Самое простое решение:** Отключить Vercel Authentication для проекта, так как:
- Статические ресурсы не могут получить query параметры автоматически
- Protection Bypass работает только для основного запроса, но не для ресурсов
- Для публичного сайта это нормально

Альтернатива: Использовать Protection Bypass через заголовок, но это требует изменений в коде загрузки ресурсов.
