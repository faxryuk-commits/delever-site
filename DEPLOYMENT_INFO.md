# Информация о деплое Delever Site

## Как работает деплой на Vercel

### Процесс деплоя:

1. **GitHub репозиторий**: `https://github.com/faxryuk-commits/delever-site.git`
2. **Ветка для Production**: `main`
3. **Автоматический деплой**: При каждом пуше в `main` ветку Vercel автоматически:
   - Клонирует репозиторий
   - Устанавливает зависимости (`npm install`)
   - Собирает проект (`npm run build`)
   - Деплоит на production

### Домены:

После подключения домена `delever.io`:

1. **Production домен**: `delever.io` (и `www.delever.io`, если настроен)
2. **Vercel домен**: `delever-site-*.vercel.app` (все еще работает)
3. **Preview домены**: Для каждого PR создается отдельный preview URL

### Важно:

- **Production деплой** происходит только при пуше в `main` ветку
- Все изменения должны быть **закоммичены и запушены** в `main`
- После пуша деплой занимает **2-5 минут**
- Vercel автоматически обновляет домен `delever.io` после успешного деплоя

## Почему изменения могут не отображаться:

### 1. Изменения не закоммичены/не запушены
**Решение**: Проверьте `git status` и убедитесь, что все изменения запушены

### 2. Деплой еще не завершился
**Решение**: Проверьте статус в Vercel Dashboard

### 3. Кэш браузера
**Решение**: 
- Откройте в режиме инкогнито
- Или жесткая перезагрузка: `Ctrl+Shift+R` (Windows) / `Cmd+Shift+R` (Mac)

### 4. Кэш CDN Vercel
**Решение**: 
- Подождите 5-10 минут
- Или сделайте редеплой в Vercel Dashboard (Redeploy)

### 5. Смотрите на старый деплой
**Решение**: 
- Убедитесь, что открываете `delever.io`, а не старый preview URL
- Проверьте, что домен правильно настроен в Vercel

## Как проверить статус деплоя:

1. Откройте Vercel Dashboard: `https://vercel.com/delever/delever-site`
2. Проверьте вкладку "Deployments"
3. Убедитесь, что последний деплой:
   - Имеет статус "Ready" (зеленая галочка)
   - Создан из последнего коммита
   - Привязан к домену `delever.io`

## Как принудительно обновить деплой:

1. В Vercel Dashboard откройте последний деплой
2. Нажмите "Redeploy" (три точки → Redeploy)
3. Или сделайте пустой коммит: `git commit --allow-empty -m "trigger redeploy" && git push`

## Проверка изменений:

После деплоя проверьте:
- Откройте `delever.io` в режиме инкогнито
- Проверьте версию файлов в DevTools (Network tab)
- Убедитесь, что загружаются новые файлы с хешами
